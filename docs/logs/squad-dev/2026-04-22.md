## [14:20] Migracao Tailwind v3.4 → v4 (correcao de divida tecnica da Story 01)

- **Contexto:** PRD/Plan/Story 01 pediram `tailwindcss@^4` + `@astrojs/tailwind@^5` simultaneamente. Combinacao impossivel: `@astrojs/tailwind@5` tem peer dep `tailwindcss@^3.0.24` e e v3-only (integration nao foi portada pro v4 e nao sera). No `npm install` da Story 01 bateu `ERESOLVE`, o implementer-haiku caiu pro `tailwindcss@^3.4` e travou via `overrides` no package.json. Log da onda 1 registrou isso como "v3.4 por compat", mas o estado final contradiz o PRD.
- **Diagnostico:** `@astrojs/tailwind` e legacy; caminho oficial do Tailwind v4 em Astro e o plugin Vite `@tailwindcss/vite` + config 100% em CSS via `@theme`.
- **Migracao executada:**
  - `site/package.json`: removido `@astrojs/tailwind@^5` e o bloco `overrides`; adicionado `@tailwindcss/vite@^4`; bump `tailwindcss: "^3.4"` → `"^4"`.
  - `site/astro.config.mjs`: removido `import tailwind from '@astrojs/tailwind'` + integration; adicionado `import tailwindcss from '@tailwindcss/vite'` + `vite: { plugins: [tailwindcss()] }`.
  - `site/src/styles/global.css`: `@tailwind base/components/utilities` → `@import "tailwindcss"`. Todo o conteudo do antigo `tailwind.config.mjs` (cores, fontes, keyframes, animacoes) migrado pra bloco `@theme` (CSS vars `--color-*`, `--font-*`, `--animate-*`). `theme('colors...')`/`theme('fontFamily...')` no base layer trocados por `var(--color-*)`/`var(--font-*)`.
  - `site/tailwind.config.mjs`: **deletado**.
  - `rm -rf node_modules package-lock.json .astro dist` + `npm install` limpo → instalou `tailwindcss@4.2.4` + `@tailwindcss/vite@4.2.4`.
- **Validacao:**
  - `npx astro build` → 1.21s, zero warnings, CSS gerado contem todas as utility classes custom (bg-page, bg-surface, bg-accent, hover:bg-accent-hover, text-ink-primary/secondary/muted, text-accent, border-rule, border-accent, font-display, font-sans, font-mono, animate-live-pulse, animate-marquee-left, animate-marquee-right).
  - `npx astro dev` fresh retornou `200 OK` na homepage sem warnings.
  - Warning `@layer base is used but no matching @tailwind base directive is present` apareceu apenas no dev server **zumbi** que ficou rodando desde 13:30 (antes da migracao). Apos `pkill -f "astro dev"` + `rm -rf .astro node_modules/.vite` + restart, warning sumiu. Causa: Vite hot-reload com state em memoria apontando pro pacote v3 antigo que eu ja tinha desinstalado.
- **Documentacao atualizada:**
  - `docs/prds/intensivo-claude-code.md` secao 2 (Stack), 3.3 (header dos tokens), 5.7 (codigo de referencia completo em @theme), 6 (deps + astro.config).
  - `docs/plans/intensivo-claude-code.md` secao "Dependencias npm" e Story 01/02 embutidas.
  - `docs/stories/intensivo-claude-code-01-scaffold-astro-em-site.md` (codigo de referencia + callout de warning).
  - `docs/stories/intensivo-claude-code-02-tokens-tailwind-traduzidos-da-fase-0.md` (reescrita completa com @theme, armadilhas do v4, criterio de aceite novo rejeitando `tailwind.config.mjs`).
- **Mapeamento v3 → v4 aplicado no site:**
  - `fontFamily.sans` → `--font-sans`. Classe `font-sans` inalterada.
  - `colors.accent.DEFAULT`/`.hover`/`.deep` → `--color-accent`/`--color-accent-hover`/`--color-accent-deep`. Classes `bg-accent`, `bg-accent-hover`, `bg-accent-deep` inalteradas.
  - `colors.ink.primary`/`.secondary`/`.muted` → `--color-ink-primary` etc. Classes inalteradas.
